---
name: create-plan
description: 탐색 완료 후 실행 계획을 문서화한다.
---

## 입력

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `issue_id` | ✅ | 참조할 이슈번호 (예: SKT-42) |

## 목적

- 명확하고 최소한의 실행 단계 정의
- 진행 상태 추적 가능한 체크리스트 생성
- 불필요한 복잡도 배제

## 계획 원칙

1. **최소화** - 명시된 요구사항만 포함
2. **모듈화** - 독립적으로 검증 가능한 단계
3. **기존 패턴 준수** - 코드베이스 컨벤션 유지

## 출력 형식

```markdown
# 실행 계획

**전체 진행률:** `0%`

## 요약
[무엇을 왜 구현하는지 1-2문장]

## 주요 결정사항
- 결정 1: [선택] - [근거]
- 결정 2: [선택] - [근거]

## 작업 목록

- [ ] 🟥 **단계 1: [이름]**
  - [ ] 🟥 세부작업 1
  - [ ] 🟥 세부작업 2

- [ ] 🟥 **단계 2: [이름]**
  - [ ] 🟥 세부작업 1
  - [ ] 🟥 세부작업 2

## 위험 요소
- [위험 1]: [대응 방안]
```

## 상태 이모지

| 이모지 | 상태 |
|--------|------|
| 🟩 | 완료 |
| 🟨 | 진행 중 |
| 🟥 | 대기 |

## 이슈 상태 업데이트

- **시작 시**: `mcp__linear__save_issue`로 이슈 status → `in_progress`
- **완료 시**: `mcp__linear__save_comment`로 코멘트 추가

## 이슈 업데이트

계획 완료 시 **반드시** Linear 이슈에 `mcp__linear__save_comment`로 상세 코멘트를 추가한다.
요약본이 아닌 계획의 **전체 내용**을 포함해야 한다. 사용자가 별도로 요청하지 않아도 항상 실행한다.

코멘트에 포함할 내용 (생략 금지 — 모든 항목 전체 작성):

```markdown
### ✅ Plan 완료 - [timestamp]

**소요 시간**: [duration]

---

## 주요 결정사항
> **탐색·사용자 확인 단계에서 결정된 모든 항목을 빠짐없이 포함한다. 근거 생략 금지.**

| 항목 | 선택 | 근거 |
|------|------|------|
| [항목1] | [선택] | [구체적 근거 — 왜 이 선택인지] |
| [항목2] | [선택] | [구체적 근거] |
| ... | ... | ... |

## 시스템/문서 구조 (해당 시)
[블록 트리, 데이터 모델, 파일 구조, 아키텍처 다이어그램 등 — 구체적으로 전체 작성]

## 작업 목록
> **모든 Phase와 세부 작업을 빠짐없이 작성한다. Phase 개수·작업 수 임의 축약 금지.**

### Phase 1: [이름]
- [ ] 세부작업 1 (구체적으로)
- [ ] 세부작업 2 (구체적으로)

### Phase 2: [이름]
- [ ] 세부작업 1
- [ ] 세부작업 2

### Phase N: [이름]
- [ ] 세부작업 1

## 위험 요소
> **발견된 모든 위험 요소를 포함한다. 1개로 축약 금지.**

| 위험 | 영향 | 대응 방안 |
|------|------|----------|
| [위험1] | [영향범위] | [구체적 대응] |
| [위험2] | [영향범위] | [구체적 대응] |
| ... | ... | ... |

**다음 단계**: execute

---
이슈: [issue_id]
```

## Orchestrator 활용

```
explore(issue_id) 완료 + 사용자 확인
    ↓
Orchestrator: plan(issue_id) 호출
    ↓
실행 계획 문서 생성
    ↓
이슈 코멘트 추가 (계획 요약)
    ↓
사용자 승인
    ↓
execute(issue_id)로 이동
```
