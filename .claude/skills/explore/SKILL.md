name: explore

---

## 입력

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `issue_id` | ✅ | 참조할 이슈번호 (예: SKT-42) |

## 목적

- 기존 코드베이스/데이터 구조 분석
- 기능/작업의 통합 방식, 의존성, 제약조건 파악
- 불명확하거나 모호한 요구사항 식별
- 명확화가 필요한 질문 목록 작성

## 수행 원칙

1. **구현하지 않는다** - 탐색과 질문 수집만 수행
2. **먼저 스스로 답한다** - 코드·문서·컨텍스트로 해소 가능한 모호성은 자문자답으로 해결, 사람에게 묻지 않는다
3. **범위를 벗어나지 않는다** - 명시된 범위 내에서만 탐색

## 소크라테스식 자문자답 (모호성 해소)

탐색 중 요구사항이 불명확하거나 설계 결정이 필요한 경우, **사용자에게 묻기 전에** 먼저 스스로 질문하고 답한다. 코드베이스·문서·기존 패턴에서 근거를 찾아 잠정 결론을 도출한 뒤, 확신도가 낮은 항목만 사용자에게 확인한다.

### 자문자답 형식

```markdown
### 자문자답 — 모호성 해소

**Q1. [모호한 사항]**
- 근거: [코드/문서/기존 패턴에서 찾은 단서]
- 잠정 답변: [결론]
- 확신도: 🟢 높음 / 🟡 중간 / 🔴 낮음

**Q2. [모호한 사항]**
- 근거: [...]
- 잠정 답변: [...]
- 확신도: 🔴 낮음 → **사용자 확인 필요**
```

### 확신도 기준

| 확신도 | 조건 | 처리 |
|--------|------|------|
| 🟢 높음 | 코드/문서에 명시적 근거 존재 | 잠정 답변으로 진행, 사용자 알림만 |
| 🟡 중간 | 기존 패턴에서 유추 가능 | 잠정 답변으로 진행, explore 결과에 명시 |
| 🔴 낮음 | 사람만 알 수 있는 정보 (사업 의도, 외부 시스템 제약 등) | **사용자에게 확인** |

> **원칙**: 🟢/🟡 항목은 사용자를 블로킹하지 않는다. 🔴 항목만 묻되, 한꺼번에 모아서 1회만 질문한다.

## 출력 형식

```markdown
## 탐색 결과

### 현재 상태
- [기존 구조/코드/데이터 요약]

### 통합 방식
- [어떻게 연결되는지]
- [의존성 목록]

### 제약조건
- [기술적 제약]
- [정책적 제약]

### 명확화 필요 항목
1. [질문 1]
2. [질문 2]
...
```

## 이슈 상태 업데이트

- **시작 시**: `mcp__linear__save_issue`로 이슈 status → `in_progress`
- **완료 시**: `mcp__linear__save_comment`로 코멘트 추가

## 이슈 업데이트

탐색 완료 시 **반드시** Linear 이슈에 `mcp__linear__save_comment`로 상세 코멘트를 추가한다.
요약본이 아닌 탐색에서 발견한 **전체 내용**을 포함해야 한다. 사용자가 별도로 요청하지 않아도 항상 실행한다.

코멘트에 포함할 내용:

```markdown
### ✅ Explore 완료 - [timestamp]

**소요 시간**: [duration]

---

## 1. 현재 상태
[기존 코드베이스/데이터/페이지 구조 전체 요약 — 섹션, 필드, 선택자 등 구체적으로]

## 2. 통합 방식 및 의존성
[어떻게 연결되는지, 외부 시스템/라이브러리/API 목록]

## 3. 제약조건
| 항목 | 제약 | 대응 방안 |
|------|------|----------|
| [항목1] | [제약] | [대응] |

## 4. 자문자답 — 모호성 해소

**Q1. [모호한 설계 결정]**
- 근거: [코드/문서에서 찾은 단서]
- 잠정 답변: [결론]
- 확신도: 🟢 높음

**Q2. [모호한 설계 결정]**
- 근거: [유추 근거]
- 잠정 답변: [결론]
- 확신도: 🔴 낮음 → **사용자 확인 필요**

## 5. 사용자 확인 필요 항목 (🔴만)
1. [질문 1 — 확신도 낮음]

**다음 단계**: plan

---
이슈: [issue_id]
```

## Orchestrator 활용

```
create-issue 완료 (issue_id: SKT-42)
    ↓
Orchestrator: explore(issue_id) 호출
    ↓
탐색 수행
    ↓
자문자답 — 모호성 자체 해소 (🟢/🟡)
    ↓
🔴 항목만 사용자에게 확인 (없으면 스킵)
    ↓
이슈 코멘트 추가 (탐색 결과 + 자문자답 포함)
    ↓
plan(issue_id)로 이동
```
